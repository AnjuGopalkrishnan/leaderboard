The metrics avaialbe in the Query Combat Platform are described below

Execution Time

Execution time refers to the time taken by the server to execute a SQL query and return the result to the client. Execution time is measured from the time the query execution starts until the result is returned to the client. It includes the time taken to fetch data from the disk, execute joins and other operations, and send the result back to the client.

Plannig Time

Planning time refers to the time taken by the query planner to generate an execution plan for a given SQL query. When a SQL query is submitted to the database, the query planner analyzes the query and generates an optimal execution plan for that query. The execution plan outlines how the database will execute the query and what indexes, joins, or other operations will be used to retrieve the data. Planning time is measured from the time the query is received by the server until the execution plan is generated.

Total Time

Total time is the sum of the execution time and the planning time.

Query Complexity

The complexity of a query is determined by counting the number of keywords, such as "JOIN", and operators used in the query. A higher score indicates a more complex query.
